# Sauvetage - Système Global pour Librairie

Bienvenue dans le dépôt principal du projet Sauvetage, un système centralisé comprenant un ERP et un site e-commerce pour une librairie.

## 📋 Vue d'ensemble

Ce dépôt sert de point central pour l'ensemble du projet. Il contient :
- La documentation générale du système
- Les liens vers les sous-projets
- Les documents administratifs, techniques et commerciaux
- L'architecture globale du système

## 🗂️ Structure du Dépôt

```
sauvetage/
├── docs/               # Documentation générale
│   ├── admin/         # Documents administratifs (cahier des charges, etc.)
│   ├── technical/     # Documentation technique (architecture, stack, etc.)
│   └── commercial/    # Documents commerciaux
├── subprojects/       # Références aux sous-projets
├── README.md          # Ce fichier
└── LICENSE            # Licence Apache 2.0
```

## 🔗 Sous-Projets

Les différents composants du système sont organisés en sous-projets distincts :

- **ERP Backend** - Système de gestion intégré pour la librairie
- **E-commerce Frontend** - Site web pour les clients
- **API Gateway** - Point d'entrée unifié pour les services
- **Base de données** - Schémas et migrations

Voir le dossier [subprojects/](subprojects/) pour plus de détails et les liens vers chaque sous-projet.

## 🚀 Démarrage Rapide

Pour débuter avec le projet :

1. Consultez le [Cahier des Charges](docs/admin/cahier-des-charges.md) pour comprendre les objectifs
2. Lisez l'[Architecture Système](docs/technical/architecture.md) pour la vue d'ensemble technique
3. Explorez les sous-projets selon vos besoins
